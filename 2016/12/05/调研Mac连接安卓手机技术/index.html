  • 什么是 adb简介

    • 安装 brew install android-platform-tools
    • 是一个开发工具，帮助安卓设备和个人计算机之间的通信。这种通信大多是在USB电缆下进行，但是也支持Wi-Fi连接。adb还可被用来与电脑上运行的安卓模拟器交流通信
    • 获取adb和fastboot的最佳方法是直接从Google下载Android SDK
    • 可以在计算机的终端程序中执行的最常用命令如下，CyanogenMod也不例外：
    • adb shell —— 该命令在计算机中提供了与设备的命令行会话相应的shell接口。在实际使用中，这使用户可以“进入”他们设备，就像使用Linux或者Mac OS X计算机一样。个别的外壳命令可以通过键入adb shell <要在设备上执行的命令>发送到设备来执行。
    • adb push <本地文件> <远程文件> —— 该命令将计算机中的<本地文件>作为<远程文件>推送到设备上。你可以在本地文件或远程文件中使用完整路径。
    • adb pull <远程文件> [<本地文件>] —— 该命令将拉取设备上的远程文件（可以包含路径）到计算机的指定目录，如果没有指定目标目录，文件会保存到命令运行时的当前目录。
    • adb logcat —— 该命令允许您实时查看设备日志。它对于调查和报告错误是很有用的。在较新版本的Android中，您可以添加-C（即adb logcat -C）以获得着色文本。
    • adb install <文件> —— 用于“侧载”应用（使用 .apk 格式）到设备上。
    • 更多有用的 命令 ，请在这里察看全部指令列表。

进程间通信

>
首先，XPC 更多关注的是实现功能某种的方式，通常采用其他方式同样能够实现。并没有强调如果不使用 XPC，无法实现某些功能。

XPC 目的是提高 App 的安全性和稳定性。XPC 让进程间通信变得更容易，让我们能够相对容易地将 App 拆分成多个进程的模式。更进一步的是，XPC 帮我管理了这些进程的生命周期，当我们需要与子进程通信的时候，子进程已经被 XPC 给运行起来了 XPC 是 OS X 下的一种 IPC (进程间通信) 技术, 它实现了权限隔离, 使得 App Sandbox 更加完备.

  • OSX10.7以后，cocoa新增加了一种新的技术，就是XPC
  • XPC在10.8以后，直接在Foundation.framework中添加了NSXPCConnection相关的类
  • 查找电脑中使用xpc 服务的应用 find /Applications -name \*.xpc
